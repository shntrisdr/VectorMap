# Serendipity Engine: The Semantic Movie Map (MVP)

## 1. プロジェクト概要

* **プロジェクト名（仮称）:** Serendipity Engine: The Semantic Movie Map
* **MVPゴール:**
  * ユーザーが指定した作品を中心に、関連作品の2Dマップを動的に生成できる。
  * マップ上で選択した2作品間の関係性を、LLMを用いて自然言語で説明できる。
  * 上記のコア体験が、ターゲットユーザーにとって価値があるか（面白い、新しい発見があると感じるか）を検証する。

## 2. ターゲットユーザー

* 新たな映画との出会い、セレンディピティを求めるユーザー。

## 3. 機能要件（MVPスコープ）

### 3.1. データ処理機能 (簡略化)
* **メタデータ収集:** 事前に収集・整形した静的なデータセット（モックデータ）を使用する。
* **特徴量ベクトル化 (最小構成):**
  * あらすじ、キーワード、ジャンル等のテキスト情報のみを特徴量として扱う。Sentence-BERTモデルを用いてベクトル化する。

### 3.2. ユーザーインターフェース（UI）機能 (最小構成)
* **入力機能:** ユーザーが起点としたい映画をテキスト検索し、候補から選択できる。
* **動的マップビュー (基本機能のみ):**
  * ユーザーの入力に応じて、関連性の高い作品群の2Dマップを動的に生成・表示する。
  * 基本的なインタラクション（ズーム、パン、作品情報のポップアップ表示）は実装する。
* **作品選択と関係性表示:** マップ上の任意の2作品を選択し、それらを結ぶベクトルを表示する。
* **関係性説明ビュー:** 選択された2作品間の関係性を説明する自然言語テキストを表示する。

### 3.3. コアエンジン機能 (核心機能に集中)
* **次元削減・マッピング:** UMAPアルゴリズムを用いて、動的に2次元マッピングを計算する。
* **関係性ベクトルの計算:** 選択された2作品の差分ベクトル `Δv` を計算する。
* **自然言語説明生成:** 差分ベクトルと「活性化エンジニアリング」等の手法を用いて、LLMによる関係性の説明を生成する。

## 4. 非機能要件 (緩和)
* **パフォーマンス:** 実用に耐えうる範囲で動作すること（厳密な目標値なし）。
* **ユーザビリティ:** シンプルなUI。
* **信頼性:** LLMのハルシネーション対策は優先せず、生成される内容の傾向を把握することを優先。
